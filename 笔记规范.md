# 笔记规范

本文件是此 Vault 的写作规范。任何工具（包括 Claude Code）在此 Vault 创建或编辑笔记时，应遵循以下约定。

---

## Frontmatter 规范

每篇笔记顶部使用以下 YAML frontmatter：

```yaml
---
tags:
  - 标签1
  - 标签2
created: YYYY-MM-DD
updated: YYYY-MM-DD
status: 草稿 | 进行中 | 已完成 | 归档
---
```

### 字段说明

| 字段 | 必填 | 说明 |
|---|---|---|
| `tags` | 是 | 至少一个标签。使用 `- 标签名` 列表格式，不要用 `[标签1, 标签2]` |
| `created` | 是 | 创建日期，格式 `YYYY-MM-DD` |
| `updated` | 否 | 最后更新日期，Claude Code 编辑时自动更新 |
| `status` | 否 | 笔记状态：草稿、进行中、已完成、归档 |

---

## 目录结构

知识工作区位于 `gmemory/` 目录，以下为工作区内的结构约定：

```
gmemory/
├── Clippings/        # Obsidian Web Clipper 保存的原始剪藏
├── 收件箱/           # remember / import 写入的碎片，待 Ingest
├── 学习/             # 学习笔记、技术文章
│   └── concepts/     # Compile 自动创建的概念页
├── 索引/             # MOC 主题索引（Compile 创建/更新）
├── 项目/             # 按项目归类
│   └── ProjectName/
├── conflict/         # Compile 检测到的语义冲突（待负责人决策）
└── 附件/             # 图片、PDF 等非 Markdown 文件
```

**双语目录支持**: 上述目录均支持英文等效名。`收件箱/` = `inbox/`，`学习/` = `notes/`，`索引/` = `index/`，`项目/` = `projects/`。Skills 会按实际情况自动检测并使用已有目录。

---

## 命名规范

- 笔记名使用中文，清晰描述内容
- 同一项目下的笔记加前缀，如 `项目A-需求`、`项目A-周报`
- 模板文件放在 `模板/` 目录

---

## 链接规范

- 使用 Obsidian 双向链接 `[[笔记名]]` 而非 Markdown 标准链接
- 指向不存在笔记的链接（占位链接）是可以接受的，Claude Code 可以为这些占位链接创建对应笔记
- MOC 索引笔记中，链接按逻辑分组排列

---

## Claude Code 操作约定

- 创建新笔记时，自动填入 frontmatter（tags 需用户确认或基于内容推断）
- 编辑已有笔记时，更新 `updated` 字段
- 不要修改 `.obsidian/` 目录下的任何文件
- 批量操作前先列出受影响文件清单，等用户确认
